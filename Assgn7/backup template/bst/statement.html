Himalaya Food Court, Tokens and Trees

Note: All code should be written within this template directory. Test cases have been provided in the folder tests, and can be run using make test.

Only a bare-bones skeleton of Makefile has been given; you will need to write the remainder yourself.

In the Himalaya Food Court, the cashier at the counter gives customers two tokens for their order. They submit one token at the food counter, and wait until their food arrives. When it does, their token number will be called, they will return the second token to verify that the order was theirs, and then pick up the food.

This is a fairly simple system, and any token based system will operate similarly. Conceptually, we need to store a collection of tokens. When a customer gives the first token, we need to add it to this collection. When the customer returns the second token, we need to be able to search through this collection, and if the token was found in the collection, it must be removed from it. Many different data structures could be used to store this collection, for example the linked lists you have already used.

We will look at another way of storing these items; in a binary search tree. Both adding and removing items to this list takes time on the order of the height of the tree, which can well be logarithmic in the number of nodes in the tree, and thus can be far more efficient than using a linked list.

Binary Search Tree

Implement a binary search tree with support for the following operations:

  • Adding a integer to the tree.
  • Removing a integer to the tree.
  • Searching for a integer to the tree.
  • Printing the values of the tree in sorted order.
  • Printing the values of the tree in a breadth-first manner.

The Token System

You will be given input as a stream of token numbers; the stream will be terminated with an input of -1. For every token number, check if the token already exists in the collection; if it does not, then add it, else remove it. After each token number, print the whole entire collection in sorted order.

Extra Credit:Compare the performance of the token system implemented using a linked list instead of the binary search tree.
