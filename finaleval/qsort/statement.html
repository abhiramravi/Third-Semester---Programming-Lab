Quick Sort

Note: All code should be written within this template directory. Test cases have been provided in the folder tests, and can be run using make test.

Quick sort is widely accepted as the fastest (hence the name) comparison sort algorithm. The key to quick sort is the following simple observation; a list is sorted if any disjoint range is ordered. At a high level, it proceeds as follows:

  1. Given a range, choose any element in it. Call it the pivot.
  2. Rearrange all the elements of the list less than the pivot to be to it's left, and similarly rearrange all elements greater than it to be on it's right.
  3. Repeat this procedure for the elements to the left of the pivot, and for the elements to the right of the pivot.

Though this description is naturally recursive, the subproblem (sorting the elements to the right or left of the pivot) can be attempted in any order. In this assignment you will use a stack (provided) to store the thus far unsorted partitions. Try to use loop invariants to partition the elements in the list, and to correctly add and remove partitions from the stack.

Sample Input/Output

Input: 5 3 4 7 3 -5
Output: -5 3 3 4 7
